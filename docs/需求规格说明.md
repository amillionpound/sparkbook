# Sparkbook（灵感簿）需求规格说明

> 版本：v0.3 需求收口稿 · 2026-07-17
> 用途：把这几天讨论的所有决策系统整理，✅=已确认，🔶=待细化，🐞=已知问题。v0.3 在 v0.2（9 项拍板）基础上，把灵感极速录入原型 v2 的 5 项交互反馈正式收口为需求（见第四节-5、第三节条目编辑规则）。

---

## 一、产品定位

个人**多端加密记事本**。日常记杂项、短期任务（可勾选划去）、账本、会议、灵感；一般只存文本。核心诉求：**随手记 + 多端同步 + 数据安全 + 能一键取走**。

---

## 二、技术架构 ✅

| 层 | 方案 |
|---|---|
| 前端 | 纯静态 HTML/CSS/JS，托管在 CloudStudio / GitHub Pages |
| 后端 | 腾讯云 SCF `sparkbook-api`（Flask 纯 JSON API，不返 HTML） |
| 存储 | 腾讯云 COS 存**密文**（明文永不出浏览器） |
| 加密 | 浏览器端 Web Crypto：主密码 PBKDF2-SHA256(15万次) → AES-256-GCM |
| 零知识边界 | SCF/COS 永不接触明文与主密码；SCF 只代理 LLM 与 ASR |

**安全约定 ✅**：忘记主密码 = 数据不可恢复（不做应急找回）。→ 所以「导出」是**必选安全网**，不是可选功能。

---

## 三、数据模型 ✅

三级结构，别混淆：

- **类型 Type（最高级，固定 5 种）**：杂项 / 任务 / 账本 / 会议 / 灵感
- **分类 Category（用户自建，正交桶）**：如"工作""个人""项目A"，跨类型复用
- **条目 Entry**：一条记录，挂在某个 Type 下，可打一个 Category 标签
- ✅ **全局编辑规则（用户 2026-07-17 明确）**：**所有记录类型均可编辑**；时间戳同时保留 `created`（创建时间）与 `updated`（修改时间），列表/详情按版面二选一或同时展示，已修改过的条目标"已编辑"

✅ **各类型字段定义（已锁定 2026-07-17）**
- **杂项**：正文（纯文本）
- **任务**：标题、完成状态（勾选划去）、**截止日期**（保留，可选填）；**不做到期提醒**
- **账本**：金额、**收/支方向**（二选一，必填）、备注、日期；**不分币种**（统一 ¥）
- **会议**：标题、正文/纪要、关联录音；**不设独立参会人字段**（需要时用户自行写进正文）
- **灵感**：正文、捕获时间

## 四、功能模块

### 1. 条目管理 ✅（原型已验证）
增删改查、按类型/分类过滤、全文搜索。
🐞 **分类交互 bug**：原生下拉在沙箱点不动 → 前端重做时改为**可点击胶囊**并彻底验证。

### 2. 任务 ✅（已锁定）
- ✅ 勾选完成、划去
- ✅ 截止日期（可选填，仅展示，**不做到期提醒**）

### 3. 账本 ✅（已锁定）
- ✅ 逐笔记账（金额 + 收/支方向 + 备注 + 日期）
- ✅ **按月汇总**（当月收入/支出/结余）
- ✅ **按分类汇总**（同为"账本"类型下，按用户分类拆分统计，如 生活 / 加油 / 停车费 各自小计）
- ❌ 不做预算阈值/超支提醒

### 4. 会议 ✅基础 / 🔶录音
- ✅ 手动记录会议内容（参会人写进正文）
- 🔶 **录音导入生成纪要**（详见第六节）

### 5. 灵感 ✅（作为 Type，非默认分类；极速录入原型 v2 已验证）
- ✅ **极速录入入口**：右下角悬浮 FAB（带数量角标）+ 快捷键 `N` 唤起底部弹层；输入后 `Enter` 立即保存并清空，支持连续速记；`Esc`/遮罩/「退出」收起（2026-07-17 原型确认手感）
- ✅ **退出交互**：原"完成"按钮改为「退出」并置于保存左侧；若退出时框内有内容，弹确认「丢弃 / 先保存再退出」，避免误丢
- ✅ **自定义分类**：分类行可内联新增（「＋分类」按钮，Enter 确认），用户自建分类持久化，与全局 Category 体系一致
- ✅ **多选导出文本**：每条可勾选，工具栏「全选 + 导出所选」→ 弹出**可编辑 textarea**（格式 `[序号].[时间][分类] 正文`），改完「复制到剪贴板」；区别于第六节的全量 MD/JSON 备份导出
- ✅ **转为普通记录**：单条或批量勾选后「转换」，选目标类型（杂项/任务/账本/会议），按类型差异补字段——
  - 杂项：无补充
  - 任务：补**截止日期**
  - 账本：补**金额(¥) + 收/支方向**
  - 会议：补**会议日期**
  - 转换后原文保留，标记 `convertedFrom:inspiration` + 转换时间
- ✅ 全记录编辑 + 双时间戳（见第三节条目定义）

### 6. 导出 ✅（安全网，必选，已锁定排版）
- ✅ **双格式**：Markdown（人读 + 喂日报助手）+ JSON（程序化导入恢复）
- ✅ 本机留存，应对"用量激增/服务停用导致数据取不出"
- ✅ **Markdown 排版规范**：**先按类型分组**（杂项/任务/账本/会议/灵感）→ **组内按分类再分组** → 组内**按日期排序**

### 7. 日报助手 ✅（内置网页，设计已详细锁定，见第五节）

### 8. 周报汇总 🔶（纳入范围，待细化）
- 拟：按周聚合当周日报/条目，生成周报初稿（复用日报助手的风格档案与两段式模型）
- 🔶 细节（周起止、聚合口径、是否也走"修改=进化"）后续开工前定

### 9. 多端同步 ✅（已锁定）
- 以 COS 为中心，**冲突处理 = 最后提交胜出**（Last-Write-Wins）
- 依据：日常以**新增记录**为主，修改既有记录的场合很少，LWW 足够；不引入版本向量/手动合并的复杂度

## 五、日报助手（设计已锁定）✅

### 5.1 两段式模型 ✅
| 动作 | 模型 | 触发频率 | 说明 |
|---|---|---|---|
| 生成日报/纪要 | `deepseek-chat` | 每天 1-2 次 | 风格套用+结构化，便宜 |
| 风格自我进化 | `deepseek-reasoner` | 显式触发+差异守卫 | 归纳可复用规则，贵但触发少 |

均开 **prompt caching**（系统提示/风格档案命中后 input 价降至 1/10）。

### 5.2 输入源（三合一，同源处理）✅
1. **流水账**：碎片时间线/粘贴 raw log → 需结构化去口语
2. **直接写"今天做了哪些事"**：中段散文，主要做风格对齐+润色
3. **联动条目**：当天的 meeting/task/账本/灵感 自动抓取
→ 三者先汇到统一「今日素材」暂存区，再走同一套生成管线。

### 5.3 生成面板 UX ✅
- AI 生成结果落**可编辑文本框**
- 两个按钮：「复制」直取；「修改」= 采纳编辑版 + 复制到剪贴板 + 触发风格进化
- **差异守卫 ✅**：只有框内内容与上次快照（AI 初稿或上一次编辑版）有差异时才跑 reasoner 进化，否则只复制不调用，成本可控。

### 5.4 风格档案（稳定层）✅（已锁定）
- **定义**：不是日报本身，而是 AI 写日报的"写作偏好记忆"，三块内容——
  1. 术语表（行业黑话：对公集市/同业集市、OceanBase oracle模式、星环TDH、经营平台埋点、厂商对接、成本管控、VP流程、绩效重算、系统画像 等）
  2. 正式发文风格要点（精简、全文可复制、无特殊格式、段间不留空行、每篇2-4点、聚焦进度/风险/资源/验收、禁"智能表格"字眼）
  3. 初稿 vs 正式稿的对照样例（初期为空，靠日常"修改"逐步积累）
- ✅ **冷启动种子** = `日报助手.txt`（现成，随应用内置，首次解锁即有完整基线：角色设定+术语表+风格要点+进化机制说明）
- ✅ **可手动编辑**（术语表、风格要点随时改）
- ✅ **进化方式** = 日常点「修改」采纳编辑版时，reasoner 归纳可复用规则回灌档案；**不做"导入样例"入口**（用户边用边修正，让 DeepSeek 慢慢拟合）
- ✅ **存储 = 加密进 COS**（随主数据同步，保证多端一致；本地做缓存副本）

### 5.5 本期背景/角色框（临时层）✅
- 独立文本框，填当篇提示补丁（"这周代班组长""报给VP""重点是项目A"）
- **默认空、每篇独立填、不保存、不进进化闭环**（避免临时背景被学成永久偏好）

### 5.6 会议纪要联动日报 ✅基础 / 🔶细化
- ✅ 按日期读 `type=meeting` 条目，先实行「必体现参会」（列"参加：标题"）
- 🔶 是否附摘要、极简会议是否省略 → 后续并入自我进化闭环动态修正

### 5.7 生成流程（勾选式，已锁定）✅
1. 打开日报面板 → 系统把当天流水账/联动条目列成**可勾选清单**
2. **默认全选**；用户手动**取消勾选**不想写进日报的条目
3. 用户在**背景框**（5.5 临时层）补当篇提示
4. 点「**生成日报**」→ 只用勾选中的素材 + 背景框内容，调 `deepseek-chat` 生成
5. 结果落可编辑文本框 → 走 5.3（复制 / 修改=采纳+复制+进化）
> 要点：生成不是自动跑，而是**用户勾选+补充后手动触发**，可控。

## 六、录音导入生成纪要 ✅方案

- **格式**：只收 **M4A**（约 1.5MB/分钟；WAV 用户自行转格式，不接）
- **链路**：浏览器选文件 → COS 分片上传（临时桶）→ SCF 调**腾讯云 ASR**（录音文件识别）拉 COS URL 转写 → LLM 生成结构化纪要 → 存回加密 `meeting` 条目 → 自动进日报 → 删临时音频
- **隐私**：接受录音离端到 ASR（唯一破零知识的主动动作；纪要仍回加密存储）
- **不做应用内录音** ✅：纯 Web 无法息屏/后台持续录，长会议用手机原生录音机导出 M4A 再导入
- 🔶 超长会议转录（>1小时）是否先分块再送 LLM 压 input？

## 七、后端 API（已上线可用）✅

SCF `sparkbook-api`：`https://1256784020-0i70k3at89.ap-guangzhou.tencentscf.com`

| 端点 | 功能 |
|---|---|
| `POST /api/ai` | `{action:generate\|evolve, messages[]}` → DeepSeek（generate=chat / evolve=reasoner） |
| `POST /api/asr/presign` | 下发 COS 预签名 PUT URL，浏览器直传 M4A |
| `POST /api/asr/transcribe` | `{key}` → 腾讯云 ASR 转写 → 清理临时音频 |
| `GET /api/health` | 健康检查（已验证 cos/deepseek 均 true） |

## 八、需求收口结论（2026-07-17 全部拍板）✅

| # | 议题 | 结论 |
|---|---|---|
| 1 | 各类型字段 | 账本分收支不分币种；会议参会人写正文（无独立字段）；任务留截止日期但不提醒 |
| 2 | 风格档案 | 定义=写作偏好记忆；种子=`日报助手.txt`；可编辑；存 COS；不做导入样例，靠日常"修改"进化 |
| 3 | 多端同步冲突 | 最后提交胜出（LWW），因以新增为主 |
| 4 | 导出 MD 排版 | 按类型分组 → 组内按分类再分组 → 组内按日期排序 |
| 5 | 账本汇总 | 做按月汇总 + 按分类汇总；不做预算提醒 |
| 6 | 任务提醒 | 不做 |
| 7 | 灵感极速录入 | ✅ 原型 v2 验证通过，交互已确认（见第四节-5） |
| 8 | 周报汇总 | 纳入范围（第四节-8），细节开工前定 |
| 9 | 日报生成流程 | 勾选式：默认全选→取消不要的→加背景框→点「生成日报」才生成（见 5.7） |

### 仅剩的小尾巴（不阻塞开工，开工前顺带定）
- 周报的周起止/聚合口径/是否走"修改=进化"
- 超长会议（>1h）转录是否分块送 LLM
- 会议纪要写进日报的详略（并入进化闭环动态修正）
